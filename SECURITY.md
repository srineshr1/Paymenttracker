# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability, please **do not** open a public issue. Instead, send a private report to the repository maintainers via GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature.

Please include:

- A description of the vulnerability
- Steps to reproduce
- Potential impact

We aim to acknowledge reports within 48 hours and provide a fix as soon as possible.

## What to expect

- Confirmation of receipt within 2 business days
- Regular updates on the fix progress
- Credit for the discovery (if desired) once the fix is published

## Scope

This security policy covers the latest release of the app and API. Older versions may not receive security patches.
